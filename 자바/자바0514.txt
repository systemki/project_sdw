쇼핑물을 운영하려고 한다. 이떄 필요한 클래스는 무엇인가?


	클래스  왜 사용 하는게
	- 정보를 편하게 관리하고 기능을 쉽게 사용하기 위해서
 	- 구성 :  필드(멤버변수), 멤버메소드, 생성자 
	- 메소드는 객체 멤버 메스드 일 떄와 클래스 멤버 메소드일 때 매개 변수가 많이 다를수 있다.


	매개변수는 객체변수 넘어 갈수 있다.

	- 객체 변수 : 각 객체마다 멤버변수 들이 있다. 
	- 클래스 변수 :  하나의 멤버변수를 모든 객체가 같이 사용 
	
	

	상속
	 - 편하게 클래스를 만들려고
	 - 표준으로 제공되는 클래스를 상속받기 위해
 	 - 다형성을 이용하면 매개변수나 멤버 변수를 다양하게 적용 할수 있어서 
	 - 다양한 클래스를 하나의 부모 클래스로 관리하기 위해서  
	    > 
	 - 부모클래스에 있는 멤버 변수/메소드를 물려 받는것
	 - 접근제한자에 따라 접근 할수, 없을 수도 있다.	(private) 
	 - 오버라이드 :  부모클래스에게 물려받은 메소드를 재정의하는 것
 	     메소드 선언이 부모 클래스의 메소드와 같은 경우 
		
	 - 오버로드 : 동일한 이름의 메소드/생성자가 여러개 있는 경우 
	     매개변수가 다른경우 	 

	상속 
	A는 B이다
	is a관계 

	포함(멤버변수)
	A는 B를 가지고 있다.
	has a관계

	생성자 
	- 생성자는 여러개 있어도 한개만 선택된다.
	- 생성자는 객체를 생성할 때 호출 한다. 객체를 통해 호출할 수 없다.
	- 생성자 이름은 클래스명과 동일
	- 생성자가 하나도 없으면 기본 생성자가 자동으로 생성된다.
	- 리턴타입이 없다 void가 없음
	- 생성자는 부모 클래스의 생성자를 호출 한다.
class A{
	public A(){}	//생성자
	public void(){}	//메소드



	this
	- 나를 나타내는 객체

	- super : 부모를 나타내는 객체
	- super() : 부모클래스 생성자
